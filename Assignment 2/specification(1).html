1. Introduction

A hospital management system is a software application that organises the daily operations of a hospital, including managing medical staff, patients, prescriptions, and surgical operations. In this assignment you will build a simplified version of such a system. The system manages three types of medical staff — Doctors, Nurses, and Surgeons — each with their own responsibilities. Not all staff can prescribe medication, and only surgeons can perform operations. These differences are modelled using interfaces.

2. System Flow

The following steps describe the complete lifecycle of the system. Read them carefully before you start coding.

Step 1. Register Medical Staff.

The administrator adds doctors, nurses, and surgeons to the system. Each staff mem ber has a unique ID, name, and salary. Each type has additional specific attributes (e.g. specialty for doctors, shift for nurses, surgery type for surgeons).

Step 2. Register Patients.

The administrator adds patients to the system. Each patient has a unique ID, name, age, and a diagnosis.

Step 3. Assign Staff to Patient.

A staff member is assigned to a patient, creating a MedicalRecord that tracks the treatment and its status.

Step 4. Prescribe Medication.

Only staff that implement the Prescribable interface (i.e. Doctor and Surgeon) may prescribe medication for a patient.

  • If the staff member is Prescribable: the prescription is recorded and a success message is printed.
  • If not (e.g. a Nurse): an error message is printed.

Step 5. Schedule an Operation.

Only staff that implement the Operable interface (i.e. Surgeon) may schedule an operation for a patient. The operation is recorded with a scheduled date and status

“Scheduled”.

Step 6. Complete an Operation.

The surgeon completes the scheduled operation. The status of the corresponding record is updated to “Completed”.

Step 7. Display Information.

The system can display all staff members, all patients, and the full medical records of a specific patient.

3. OOP Concepts

This assignment is designed to practise the following OOP concepts. Read this section carefully — it explains why each concept is used in the system.

  • Abstract Class — MedicalStaff. All staff share common attributes (ID, name, salary) and a common abstract method getDuties(). Because different types of staff have different duties, this method cannot be implemented in the base class — each subclass must provide its own implementation. This enforces a contract without providing a default behaviour.
  • Inheritance. Doctor, Nurse, and Surgeon all extend MedicalStaff. They inherit the common fields and each adds its own specific attributes and overrides getDuties().
  • Interface 1 — Prescribable. Defines the ability to prescribe medication. Implemented by Doctor and Surgeon. Nurse does not implement it, reflecting the real-world rule that nurses cannot prescribe.

3

  • Interface 2 — Operable. Defines the ability to perform surgical operations. Implemented only by Surgeon. This demonstrates that a class can implement multiple interfaces simultaneously.
  • Polymorphism. The system stores all staff in a single MedicalStaff[] array. At runtime, instanceof checks determine whether a staff member is Prescribable or Operable before calling the corre- sponding interface methods.

Important Note

Key insight about interfaces.

A class can implement multiple interfaces but extend only one class. Surgeon extends MedicalStaff and simultaneously implements both Prescribable and Operable. This is the power of interfaces: they allow unrelated capabilities to be added to a class independently.

4. System Entities

The system manages the following entities:

Entity Description
MedicalStaff Abstract base class for all hospital staff. Holds staffId, name, and salary. Declares the abstract method getDuties().
Doctor Extends MedicalStaff, implements Prescribable. Has specialty and years of experience.
Nurse Extends MedicalStaff. Has shift type and department. Does not imple- ment any interface.
Surgeon Extends MedicalStaff, implements both Prescribable and Operable. Has surgery type and operating room number.
Patient A patient registered in the hospital. Has patientId, name, age, and diag- nosis.
MedicalRecord Created when a staff member is assigned to a patient. Stores the staff, patient, treatment details, and status.

The two interfaces used by the system are:

Interface Implemented by Methods
Prescribable Doctor, Surgeon prescribeMedication(Patient, String), getPrescriptions()
Operable Surgeon only scheduleOperation(Patient, String), completeOperation(Patient)
